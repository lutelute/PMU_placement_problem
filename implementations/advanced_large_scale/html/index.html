発展的内容：大規模系統のPMU配置

合成ネットワーク（送配電混在）でスケーリング特性を分析

なぜ大規模系統が重要か

実際の電力系統は，数千から数万のバスで構成される大規模ネットワークである． 例えば，北米東部連系系統は約60,000以上のバスを有し，日本の基幹系統でも数千バス規模に達する． PMU配置問題は本質的にNP困難（最小支配集合問題の一般化）であるため， 整数線形計画法（BILP）による厳密解法は，バス数 n が1,000を超えると計算時間が急激に増加し実用的でなくなる．

本デモでは，大規模系統に対応するための近似アルゴリズム（貪欲法）を実装し， 最大20,000バスの合成ネットワーク上でPMU配置を実行する． また，送電系統と配電系統の構造的差異がPMU配置率に与える影響を分析する．

貪欲アルゴリズムによるPMU配置

大規模系統では，BILP（二値整数線形計画法）の代わりに貪欲集合被覆アルゴリズムを使用する． このアルゴリズムは最適解の ln(n) 倍以内の近似保証を持ち，O(n · dmax) の計算量で動作する．

貪欲アルゴリズム：
1. 各バス i に対し，観測可能集合 Si = {i} ∪ N(i) を定義
2. 未観測バス集合 U ← {1, 2, ..., n}
3. U ≠ ∅ の間：
    j* = argmaxj |Sj ∩ U| を選択（最も多くの未観測バスをカバー）
    PMU集合に j* を追加，U ← U ∖ Sj*

このアルゴリズムの計算量は O(n · dmax) であり，dmax はグラフの最大次数である． 20,000バスの系統でもブラウザ上で2秒以内に解を得ることができる．

スケーリング法則

グラフの平均次数を d̄ とすると，PMU配置数は以下のスケーリング則に従う：

NPMU ≈ n / (d̄ + 1)

これは各PMUが自身と隣接バスを観測できるため，平均して (d̄ + 1) バスをカバーすることに対応する．

重要な知見：配電系統（放射状構造，d̄ ≈ 2）は， 送電系統（メッシュ構造，d̄ ≈ 4–6）と比較して， バスあたりのPMU設置率が約1.5–2倍高くなる． 実系統は配電バスが大多数を占めるため，PMU配置コストの主要因は配電系統の構造に依存する．

実装上の工夫

  • 型付き配列（Typed Arrays）：Float32Arrayで座標を，Uint8Arrayでフラグを管理し，メモリ効率と速度を改善
  • 非同期分割計算：setTimeout(0) で計算を分割し，UIフリーズを防止
  • Canvas描画：SVGは数千ノードで性能劣化するため，HTML5 Canvasを使用
  • 階層的レイアウト：1,000ノード以下は力学モデル，それ以上はグリッド/階層配置
  • 密度ヒートマップ：1,000ノード超では個別描画の代わりにセル密度で可視化
