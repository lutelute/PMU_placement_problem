Paper 4: Unified PMU Placement for Observability and Bad Data Detection

B. Gou & R. G. Kavasseri (2014) — IEEE Trans. Power Systems, Vol. 29, No. 6

DOI: 10.1109/TPWRS.2014.2307577

1. 論文の背景と目的

従来のPMU配置問題は、観測性（Observability）の確保のみを目的としていました。 しかし、実際の電力系統ではPMUだけでなく、従来型の計測器（潮流計測・注入計測）が既に設置されています。 Gou & Kavasseri (2014) は、これらの従来計測を活用しつつ、 不良データ検出（Bad Data Detection）能力も同時に保証する 統一的なPMU配置手法を提案しました。

2. 3つのアルゴリズム

Algorithm I: ヤコビアンベースの観測性配置

従来のBILP（Binary Integer Linear Programming）は、全バスに対して制約を設定するため、 問題サイズが大きくなります。本論文では、ヤコビアン行列の LDLT 分解を用いて、 従来計測で既に観測可能なバスを事前に特定し、PMU配置が必要なバスのみに問題を縮小します。

手順:
1. 従来計測のヤコビアン行列 H を構成
2. HTWH の LDLT 分解を実行
3. D の対角要素が 0 のバス = 未観測バス（PMU候補）
4. 未観測バスのみを対象に縮小BILPを解く

利点: 問題サイズが大幅に縮小され、計算効率が向上

Algorithm II: 臨界計測の除去

臨界計測（Critical Measurement）とは、それを除去するとシステムの観測性が失われる計測のことです。 臨界計測が存在すると、不良データ検出が不可能になります（冗長性がないため）。

臨界計測の問題点: 臨界計測に不良データが混入しても、検出する方法がない。 電力系統の状態推定の信頼性を確保するには、全ての臨界計測を排除する必要がある。

Algorithm II では、ヤコビアン行列の列ランクを用いて臨界計測を特定し、 追加のPMU配置によりこれらを非臨界化します。

Algorithm III: 統一的反復手順

Algorithm I と II を組み合わせた反復的な統一手順です。

反復手順 (v = イテレーション番号):

v = 0: Algorithm I を実行 → 観測性を確保
v = 1: Algorithm II を実行 → 臨界計測を特定・除去
v = 2: 再度 Algorithm II → 臨界計測ペアを除去
...
収束するまで繰り返す

収束条件: 新たな臨界計測が検出されなくなった時点で終了

3. ヤコビアンベースの利点

なぜヤコビアンアプローチが有効か:
  • 従来計測（潮流・注入）の情報を自然に取り込める
  • LDLT 分解により未観測バスを効率的に特定
  • 問題サイズの縮小により、大規模系統でも実用的
  • 臨界計測の特定にもヤコビアン行列を活用可能

4. 不良データ検出とは

状態推定において、計測値に誤差（不良データ）が混入する場合があります。 不良データ検出は、残差分析（通常は最大正規化残差テスト）により 異常な計測値を特定するプロセスです。

検出が可能であるための条件は、計測の冗長性です。 つまり、各状態変数が複数の独立な計測で推定できる必要があります。 臨界計測が存在すると、その計測に対応する状態は1つの計測のみに依存するため、 不良データの検出が原理的に不可能になります。

配置モード PMU数 PMU位置 説明
標準BILP（従来計測なし） 4 {2, 6, 7, 9} PMUのみで完全観測性を達成
従来計測あり (v=0) 1 {13} 24潮流+8注入計測で観測性ギャップが大幅縮小
統一配置（観測性+不良データ検出） 4 (5ユニット) {7, 9, 11, 13} 臨界計測を全て排除
重要な知見: 従来計測を考慮すると観測性のみなら1台のPMUで十分ですが、 不良データ検出能力を確保するには4箇所（5ユニット）が必要です。 バス11には2ユニットが配置されます。
